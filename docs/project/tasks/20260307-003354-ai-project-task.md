# AI自动化测试工具主要技术选型任务说明

## 目标

在上一轮总体架构设计基础上，给出主要技术栈选型方案，覆盖控制面、执行面、AI 编排、记忆系统、前端、数据存储、消息队列、可观测性和部署基础设施，形成后续实现阶段可直接采用的技术决策基线。

## 范围

- 明确控制面、agent、Web 执行、API 执行、AI 服务、记忆系统的主要技术选型。
- 说明每项技术的适用原因、替代方案和当前不选的理由。
- 给出 Phase 1 / Phase 2 的落地建议，避免一次性引入过多组件。
- 补齐本次任务的测试计划、测试报告和测试举证文档。

## 验收标准

- 技术选型文档完整覆盖以下主题：
  - 服务器控制面技术栈
  - Web Console 技术栈
  - 跨平台 agent 技术栈
  - Web 测试执行技术栈
  - API 测试执行技术栈
  - AI 编排与记忆系统技术栈
  - 数据存储、消息队列、对象存储、缓存与可观测性技术栈
  - 部署与插件扩展技术栈
- 选型理由具体，可支撑实现阶段决策。
- 文档校验和契约校验通过。

## 约束

- 保持与上一轮架构设计结论一致：[20260307-001530-ai-design-task.md](/home/jianghua519/ai-web-testing-platform-v2/docs/design/tasks/20260307-001530-ai-design-task.md)。
- 不为了“技术炫技”引入过多基础设施，优先选择能支撑 Phase 1 落地的组合。
- 文档使用中文，结论要明确，不写模糊的“都可以”。
