# 远程step控制协议测试计划

## 测试范围

- `HttpStepController` 的请求/响应决策逻辑
- `createStepControllerFactoryFromEnv()` 的环境变量装配逻辑
- `WebJobRunner` 的 job 级 controller 解析逻辑
- 内部控制接口契约和中文文档

## 关键风险

- worker 仍然使用静态 controller，导致 job metadata 丢失
- 控制端返回 `pause` / `replace` 时，worker 没有正确重试或应用替换 step
- OpenAPI 契约与控制器实际行为不一致
- 文档没有记录真实运行验证限制

## 测试项

1. 运行 `npm run typecheck`
2. 运行一次真实流：
   - 本地结果接收端收 step 事件
   - 本地控制端收 step 决策请求
   - step1 完成后允许远程 patch step2
3. 检查最终访问 URL 是否变成远程改写后的值
4. 运行文档校验
5. 运行契约校验
6. 检查容器运行入口是否缺失

## 通过标准

- 类型检查通过
- 真实运行流中控制端至少收到一次 `pause` 和一次 `replace` 决策请求/响应链路
- 结果接收端收到两条 step 结果和一条 job 结果
- step2 最终访问 URL 为远程改写后的 URL
- 文档校验和契约校验通过
