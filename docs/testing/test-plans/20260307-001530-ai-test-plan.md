# AI自动化测试工具架构设计测试计划

## 测试范围

验证本次架构设计文档是否完整覆盖以下内容：

- 服务器部署与控制面设计
- Windows / Linux / macOS / 容器 agent 设计
- Web 与 API 测试能力
- 扩展机制
- 从需求到测试报告的流程闭环
- AI-native 与记忆系统设计
- Web step 级控制模型

## 覆盖风险

- 架构文档未覆盖用户明确提出的关键能力。
- 文档之间追溯关系不完整。
- 校验脚本未通过，导致交付不完整。
- 设计描述过于抽象，无法指导后续实现。

## 测试项

1. 检查项目任务说明、设计说明、测试计划、测试报告、测试举证五类文档是否存在。
2. 检查设计文档是否显式覆盖部署、agent、Web/API、扩展、AI 和记忆、step 控制。
3. 运行 `bash ./scripts/validate_docs.sh`。
4. 运行 `bash ./scripts/validate_contracts.sh`。
5. 检查索引文档是否已包含本次任务条目。

## 通过标准

- 五类交付文档齐全且元数据完整。
- 设计文档覆盖本次任务的全部核心需求。
- 两个校验脚本都执行通过。
- 测试报告和证据记录可追溯到本次任务。
